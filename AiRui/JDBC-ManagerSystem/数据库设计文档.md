# 商店库存管理系统 - 数据库设计文档

## 1. 设计原则

- **规范化**：遵循数据库设计的规范化原则，确保数据的一致性和完整性
- **主键设计**：使用自增主键，确保数据的唯一性
- **外键约束**：建立适当的外键关系，保证数据的关联性
- **索引优化**：为常用查询字段建立索引，提高查询性能
- **时间戳**：添加create_time和update_time字段，记录数据的创建和更新时间

## 2. 数据库表结构

### 2.1 用户表 (user)

| 字段名 | 数据类型 | 约束 | 描述 |
|-------|---------|------|------|
| user_id | INT | PRIMARY KEY, AUTO_INCREMENT | 用户ID |
| username | VARCHAR(50) | NOT NULL, UNIQUE | 用户名 |
| password | VARCHAR(100) | NOT NULL | 密码（加密存储） |
| real_name | VARCHAR(50) | NOT NULL | 真实姓名 |
| role | VARCHAR(20) | NOT NULL | 角色（admin/manager/staff） |
| status | INT | NOT NULL DEFAULT 1 | 状态（1-启用，0-禁用） |
| create_time | DATETIME | NOT NULL DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| update_time | DATETIME | NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP | 更新时间 |

**索引**：
- 唯一索引：username
- 普通索引：role, status

### 2.2 产品表 (product)

| 字段名 | 数据类型 | 约束 | 描述 |
|-------|---------|------|------|
| product_id | INT | PRIMARY KEY, AUTO_INCREMENT | 产品ID |
| product_name | VARCHAR(100) | NOT NULL | 产品名称 |
| category | VARCHAR(50) | NOT NULL | 产品分类 |
| unit | VARCHAR(20) | NOT NULL | 计量单位 |
| price | DECIMAL(10,2) | NOT NULL | 单价 |
| supplier_id | INT | NOT NULL, FOREIGN KEY | 供应商ID |
| status | INT | NOT NULL DEFAULT 1 | 状态（1-启用，0-禁用） |
| create_time | DATETIME | NOT NULL DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| update_time | DATETIME | NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP | 更新时间 |

**索引**：
- 普通索引：product_name, category, supplier_id, status

### 2.3 供应商表 (supplier)

| 字段名 | 数据类型 | 约束 | 描述 |
|-------|---------|------|------|
| supplier_id | INT | PRIMARY KEY, AUTO_INCREMENT | 供应商ID |
| supplier_name | VARCHAR(100) | NOT NULL | 供应商名称 |
| contact_person | VARCHAR(50) | NOT NULL | 联系人 |
| phone | VARCHAR(20) | NOT NULL | 联系电话 |
| address | VARCHAR(200) | NOT NULL | 地址 |
| status | INT | NOT NULL DEFAULT 1 | 状态（1-启用，0-禁用） |
| create_time | DATETIME | NOT NULL DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| update_time | DATETIME | NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP | 更新时间 |

**索引**：
- 普通索引：supplier_name, contact_person, phone

### 2.4 库存表 (inventory)

| 字段名 | 数据类型 | 约束 | 描述 |
|-------|---------|------|------|
| inventory_id | INT | PRIMARY KEY, AUTO_INCREMENT | 库存ID |
| product_id | INT | NOT NULL, FOREIGN KEY | 产品ID |
| quantity | INT | NOT NULL | 库存数量 |
| min_stock | INT | NOT NULL DEFAULT 0 | 最低库存预警 |
| last_update_time | DATETIME | NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP | 最后更新时间 |

**索引**：
- 唯一索引：product_id
- 普通索引：quantity, min_stock

### 2.5 库存交易表 (inventory_transaction)

| 字段名 | 数据类型 | 约束 | 描述 |
|-------|---------|------|------|
| transaction_id | INT | PRIMARY KEY, AUTO_INCREMENT | 交易ID |
| product_id | INT | NOT NULL, FOREIGN KEY | 产品ID |
| transaction_type | VARCHAR(10) | NOT NULL | 交易类型（in-入库，out-出库） |
| quantity | INT | NOT NULL | 交易数量 |
| unit_price | DECIMAL(10,2) | NOT NULL | 交易单价 |
| total_amount | DECIMAL(12,2) | NOT NULL | 交易总金额 |
| operator_id | INT | NOT NULL, FOREIGN KEY | 操作人ID |
| transaction_time | DATETIME | NOT NULL DEFAULT CURRENT_TIMESTAMP | 交易时间 |
| remark | VARCHAR(200) | | 备注 |

**索引**：
- 普通索引：product_id, transaction_type, transaction_time, operator_id

## 5. 数据库配置

**config/db.properties**：

```properties
# 数据库驱动
db.driver=com.mysql.cj.jdbc.Driver
# 数据库连接URL
db.url=jdbc:mysql://localhost:3306/store_inventory?useUnicode=true&characterEncoding=utf8&useSSL=false&serverTimezone=Asia/Shanghai
# 数据库用户名
db.username=root
# 数据库密码
db.password=123456
```

## 6. 注意事项

1. **安全性**：密码必须使用BCrypt等安全算法加密存储，禁止明文存储密码。
2. **性能**：为常用查询字段建立索引，提高查询性能。
3. **一致性**：使用事务确保数据操作的原子性和一致性。
4. **可维护性**：定期备份数据库，优化数据库性能。
5. **扩展性**：设计时考虑未来业务扩展的需求，预留必要的字段和表结构。
